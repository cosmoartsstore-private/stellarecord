2. Navigation Structure

アプリルートは左サイドバーとメインペインで構成する。Archive 系モーダルは Analyze から開く。
App
  ├─ Sidebar
  │   ├─ Launcher
  │   ├─ Analyze
  │   └─ Database
  ├─ Main Section
  │   ├─ RegistrySection
  │   ├─ AnalyzeSection
  │   └─ DatabaseSection
  └─ Global Modals
      ├─ RegisterAppModal
      ├─ ArchiveSelectorModal
      ├─ LogViewerModal
      ├─ PolarisCleanupModal
      └─ CreditModal

3. Common Layout Rules

AreaDefinition
Sidebar主要セクションの切替と現在位置表示を担う。
Section header画面名、概要、主要アクションを配置する。
Content panelカード、テーブル、進捗、ログビューアを用途別に配置する。
ToastIPC 成功・失敗、取り込み完了、保存結果を通知する。
Modalファイル選択、ログ表示、削除確認など一時操作を閉じ込める。

4. Screen Layouts

Launcher

登録アプリのカード/リストを表示し、起動と管理を行う。

┌ Sidebar ┐┌ Launcher header: register / view mode ┐
│ Analyze │├───────────────────────────────────────┤
│ DB      ││ App list or app cards                 │
│         ││ [icon] name path actions              │
└─────────┘└ Toast area                            ┘

Analyze

archive 容量、取り込み、ログ閲覧、元ログ削除をまとめる。

┌ Analyze header: import / cleanup / settings ┐
├ Storage meter                               ┤
├ Import progress / event state               ┤
├ Archive list summary                         │
└ Actions: restore / open viewer / cleanup     ┘

Database Preview

テーブル選択と検索結果を表示する。

┌ Database header ┐
├ Table selector / search / paging             ┤
├ Result table                                  │
│ col | col | col | ...                        │
└ Row count / query status                      ┘

Log Viewer Modal

plain log または compressed log をストリーム表示する。

┌ Log viewer: file name / close ┐
├ Toolbar: status / cancel      ┤
├ Virtualized log lines         │
│ 12:00 line text...            │
└ Progress / error              ┘

5. Modal and Dialog Layouts
